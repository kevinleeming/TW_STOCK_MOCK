# 台股分析工具組

本專案包含兩套獨立工具：

1. **`main.py`**：ML 回測工具 —— 從成交量前50大股票中，訓練漲跌預測模型並回測績效（詳見下方「一、ML回測工具」）
2. **`screener_main.py`**：選股評分模型 —— 依籌碼面 + 布林通道 + 葛蘭碧法則，對全市場股票評分，選出約15檔優質股（詳見下方「二、選股評分模型」）

兩者共用 `data_fetch.py`（股票清單/歷史股價）與 `config.py`（參數設定），可獨立執行。

## 安裝
```bash
pip install -r requirements.txt
```
若沒有安裝 scikit-learn，`main.py` 會自動改用內建的 NumPy 版隨機森林（僅供備援，速度較慢）。
`screener_main.py` 不需要 scikit-learn。

# 一、ML 回測工具 (main.py)

## 功能流程
1. 從 TWSE（上市）與 TPEx（上櫃）開放資料 API 取得全部台股清單
2. 抓取近一個月（約22個交易日）每日成交量，篩選出成交量前 50 大股票
3. 抓取這 50 檔股票近 2 年歷史日線資料，計算技術指標特徵，並建立「未來 N 日後是否上漲」的標籤
4. 依時間序先後切分 訓練集 / 驗證集 / 測試集（避免用未來資料訓練 - look-ahead bias）
5. 訓練隨機森林分類模型，在驗證集上做回測（等權重投資組合，含交易成本）
6. 檢查驗證集績效是否達到預設門檻，未達標會自動調整模型參數重新訓練，最多 5 輪
7. 用「最終選定」的設定，在從未參與訓練或調參的測試集上做最後一次驗證，產出報告

## 使用方式
```bash
python main.py            # 正式模式：連線抓取真實資料
python main.py --mock     # 測試模式：使用模擬資料驗證流程（不需要網路）
```
結果輸出到 `report/`：`top50_stocks.csv`、`report.md`、`run_log.json`、`test_backtest_detail.csv`、`equity_curve.png`

## 回測方法論重點
由於預測目標是「未來 N 日」報酬，同一檔股票相鄰交易日的預測區間會互相重疊，若直接複利
相乘會嚴重高估報酬。程式將每筆訊號的 N 日報酬換算為「等效日報酬」，再依實際日曆日期
做等權重加總計算資金曲線，避免灌水。迭代調參只用驗證集，測試集僅在最後評估一次。

# 二、選股評分模型 (screener_main.py)

## 功能流程
1. 取得全部台股清單 + 最新收盤日（或最近交易日）全市場快照（收盤價、成交量）
2. 依成交量篩出候選池（預設前300大，效能考量的兩階段篩選，非最終結果）
3. 抓取候選池個股的日K歷史，計算技術指標
4. 抓取全市場籌碼資料：TDCC集保股權分散表（大戶/股東人數，每週）、三大法人買賣超（每日）、
   融資融券餘額（每日）
5. 逐檔計算 6 個子分數，加總為 0~100 總分
6. 選出總分前 15 檔，輸出 `report.md`（含每檔的選股理由）與 `report.html`（簡單表格 UI）

## 使用方式
```bash
python screener_main.py            # 正式模式：連線抓取真實資料
python screener_main.py --mock     # 測試模式：使用模擬資料驗證流程（不需要網路）
```
結果輸出到 `screener_report/`：`all_candidates_scores.csv`（候選池全部評分）、
`top_picks.csv`（前15檔）、`report.md`、`report.html`（用瀏覽器打開即可看表格）

## 評分機制（總分100，規則式、可重現，方便未來回測）

### 籌碼面 50分
| 子項 | 配分 | 判斷邏輯 |
|---|---|---|
| 大戶持股比例趨勢 | 20 | 股價 >= 1000元看「百張大戶」，否則看「千張大戶」；持股比例近幾週上升給高分 |
| 股東人數(散戶)趨勢 | 10 | 總股東人數下降（散戶籌碼流出、籌碼集中）給高分 |
| 三大法人買賣超 | 10 | 近10個交易日累計買賣超占同期成交量比例，買超給高分 |
| 融資餘額變化 | 10 | 近10個交易日融資餘額下降（籌碼變乾淨）給高分 |

### 布林評級 30分
依「帶寬(窄<5% / 正常 / 寬>=20%)」x「股價位置(上/中/下軌)」x「是否剛開喇叭(帶寬急速擴張)」
x「三大法人買賣方向(作為主力進出的替代指標)」綜合判斷，對應到使用者提供的5個範例情境
（如「帶寬窄+開喇叭+沿上軌+法人買」給最高分；「大帶寬+法人賣+股價在上軌」給低分示警）。

### 乖離 / 葛蘭碧法則 20分
依月線(20日均線)走勢與股價關係，判斷屬於葛蘭碧5大買賣點規則中的哪一種
（突破均線/拉回撐住=買點；乖離過大/跌破/往下反彈=賣點）。

## 重要限制與假設（請詳閱）

1. **60分鐘K線 → 改用日K線**：台股目前沒有免費官方API可取得個股的分鐘級歷史資料，
   已與使用者確認改用日K線計算 60MA / KD(60,3,3) / 布林通道等指標，運算邏輯相同，
   只是時間單位從「60分鐘」換成「日」。
2. **分點券商賣超 → 改用三大法人+融資融券近似**：分點資料通常僅存在於付費看盤軟體，
   本模型不抓取這類資料，改用官方免費的三大法人買賣超與融資融券餘額來近似「主力進出」。
3. **TDCC集保股權分散表為「週」資料，且只提供最新一期快照**：本模型採「快照累積」設計，
   每次執行時把當週快照存進本地歷史檔 (`cache/tdcc_weekly_history.csv`)，需要連續執行
   數週才能算出真正的大戶/股東人數趨勢。剛開始使用（本地歷史<2週）時，這兩項子分數
   會回傳中性值，並在報告中註明，**不會捏造趨勢數字**。建議每週執行一次以持續累積。
4. **候選池為兩階段篩選**：先用成交量篩出前300大做候選（效能考量），而非對全市場
   1700+ 檔股票都做深入分析，最終15檔是從這個候選池中選出。
5. **API 欄位格式**：TDCC / TWSE / TPEx 的欄位名稱如有調整，程式會印出解析失敗訊息，
   請對照 `chip_data.py`／`data_fetch.py` 檔頭註解的網址確認回傳格式。
6. **本工具僅供研究與教育用途，不構成投資建議**。評分機制是規則式、非機器學習驗證過的
   模型，僅反映歷史籌碼與技術指標，不保證未來股價表現，實際交易請自行評估風險。

## 可調整參數
所有門檻集中在 `config.py` 檔案下半部「選股評分模型專用參數」區塊，包含候選池大小、
大戶門檻、觀察期間長度、布林帶寬門檻、各子項配分等，方便之後微調與回測。
